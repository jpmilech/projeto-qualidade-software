# Qualidade em Metodologias Ágeis – LocalEats

## Introdução

Neste documento é apresentada uma análise do processo de desenvolvimento utilizado pela equipe do projeto LocalEats sob a perspectiva das metodologias ágeis. São avaliadas as práticas atualmente utilizadas, propostas melhorias para o processo e definidos critérios de Definition of Ready (DoR) e Definition of Done (DoD), visando aumentar a organização, a colaboração e a qualidade do desenvolvimento.

## 1. Análise de Práticas Ágeis no Processo

A tabela abaixo apresenta uma análise das principais práticas ágeis aplicadas ao processo de desenvolvimento da equipe do LocalEats.

| Prática | Existe no processo? | Como é aplicada atualmente? | Pode ser melhorada? |
|---------|----------------------|-----------------------------|---------------------|
| Planejamento iterativo | Sim | A equipe planeja as atividades antes do desenvolvimento de cada PBL. | Sim. O planejamento pode ser mais detalhado e revisado com maior frequência. |
| Priorização de funcionalidades | Sim | As funcionalidades são desenvolvidas conforme a prioridade definida pelo professor e pela equipe. | Sim. Pode ser utilizada uma priorização baseada em valor e impacto para o usuário. |
| Entregas incrementais | Sim | Cada PBL representa uma entrega parcial do projeto. | Sim. As entregas podem ser menores e mais frequentes. |
| Feedback frequente | Parcialmente | O feedback ocorre principalmente após a conclusão das atividades. | Sim. Reuniões rápidas durante o desenvolvimento podem melhorar a comunicação. |
| Trabalho colaborativo | Sim | Os integrantes dividem as atividades e compartilham os resultados. | Sim. A colaboração pode ser ampliada com revisões de código e maior integração entre os membros. |
| Controle visual das atividades | Parcialmente | O acompanhamento é feito de forma simples pelo grupo. | Sim. Um quadro Kanban pode facilitar o acompanhamento das tarefas. |
| Melhoria contínua | Sim | A equipe busca corrigir problemas identificados nas atividades anteriores. | Sim. A realização de retrospectivas ao final de cada entrega pode fortalecer esse processo. |

### Conclusão

O processo utilizado pela equipe já apresenta diversas características das metodologias ágeis, como trabalho colaborativo, planejamento das atividades e entregas incrementais. Entretanto, ainda existem oportunidades de melhoria relacionadas ao feedback contínuo, ao acompanhamento visual das tarefas e à realização de retrospectivas periódicas, tornando o desenvolvimento mais organizado e eficiente.

## 2. Propostas de Melhoria Ágil

Com base na análise realizada, a equipe identificou algumas melhorias que podem tornar o processo de desenvolvimento mais organizado, colaborativo e eficiente.

| Melhoria Proposta | Metodologia Relacionada | Benefício Esperado |
|-------------------|-------------------------|--------------------|
| Utilizar um quadro Kanban para acompanhar as atividades do projeto | Kanban | Maior organização e visibilidade das tarefas em andamento. |
| Realizar reuniões rápidas semanais para acompanhar o andamento das atividades | Scrum (Daily Meeting adaptada) | Melhor comunicação entre os integrantes e identificação rápida de dificuldades. |
| Realizar revisão de código antes da entrega das funcionalidades | XP (Pair Programming / Code Review) | Redução de erros e melhoria da qualidade do código. |
| Fazer uma retrospectiva ao final de cada PBL para discutir pontos positivos e melhorias | Scrum (Sprint Retrospective) | Promover a melhoria contínua do processo de desenvolvimento. |

## 3. Definition of Ready (DoR)

Para que uma funcionalidade possa iniciar seu desenvolvimento no projeto LocalEats, ela deve atender aos seguintes critérios:

- O requisito está claramente descrito.
- Os critérios de aceitação foram definidos.
- A prioridade da funcionalidade foi estabelecida.
- A equipe compreendeu o que deve ser desenvolvido.
- Não existem dúvidas ou impedimentos para iniciar o desenvolvimento.

## 4. Definition of Done (DoD)

Uma funcionalidade será considerada concluída quando atender aos seguintes critérios:

- O desenvolvimento foi finalizado.
- Todos os testes previstos foram executados com sucesso.
- Os critérios de aceitação foram atendidos.
- A documentação foi atualizada, quando necessário.
- A funcionalidade foi revisada e aprovada pela equipe antes da entrega.

## Conclusão

A aplicação de práticas ágeis contribui para um processo de desenvolvimento mais organizado, colaborativo e eficiente. A utilização de conceitos como Scrum, XP, Kanban, Definition of Ready e Definition of Done ajuda a reduzir falhas, melhorar a comunicação entre os integrantes da equipe e aumentar a qualidade das entregas realizadas no projeto LocalEats.
